## What is a Postmortem?
> A **postmortem** is a written analysis of a production incident that explains **what happened, why it happened, how it was fixed, and how to prevent it in the future**, without blaming individuals.

### Postmortems

This directory contains postmortem documents for production incidents.

Postmortems help teams:  
- Learn from failures  
- Improve system reliability  
- Prevent repeat incidents  
- Build operational maturity

### Principles
- No blame culture  
- Focus on systems, not people  
- Clear action items
